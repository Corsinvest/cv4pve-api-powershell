cv4pve-api-powershell

about_cv4pve-api-powershell

SHORT DESCRIPTION

PowerShell module for managing Proxmox VE via its REST API, similar to VMware PowerCLI.

LONG DESCRIPTION

The cv4pve-api-powershell module enables system administrators and developers to manage and automate Proxmox VE environments using PowerShell.

It provides a comprehensive set of cmdlets that wrap the Proxmox REST API, allowing operations such as VM and container management, node monitoring, backup handling, and storage inspectionâ€”all from PowerShell.

This module serves as the PowerCLI equivalent for Proxmox VE: - While PowerCLI facilitates VMware vSphere automation via PowerShell, - cv4pve-api-powershell offers similar capabilities for Proxmox VE environments.

Key features include: - Easy to learn and use - Supports multiple response types: JSON, PNG, EXTJS, HTML, TEXT - Full class and method generation from documentation - Indexed parameters support (e.g., -ScsiN, -IdeN, -NetN) - Utilities such as ConvertFrom-PveUnixTime, Wait-PveTaskIsFinish, Get-PveTaskIsRunning - Direct API access using Invoke-PveRestApi - High Availability support with Connect-PveCluster - Cross-platform compatibility (Windows, Linux, macOS) - Installation via PowerShell Gallery or manual download - No installation required on Proxmox VE - Open Source under GPL-3.0 license - Supports API tokens and two-factor authentication

REQUIREMENTS

  • PowerShell 6.0 or higher
  • Network connectivity to the Proxmox VE server
  • API access enabled on Proxmox VE

EXAMPLES

Install the module from PowerShell Gallery:

Install-Module -Name Corsinvest.ProxmoxVE.Api

Connect to a Proxmox server:

Connect-PveCluster -Credentials (Get-Credential) -HostsAndPorts 192.168.1.10:8006

List virtual machines on a specific node:

Get-PveVm -Node pve-node1

NOTE

This module communicates with Proxmox VE via HTTPS using its REST API. It does not require installation on the Proxmox server and can be used from any system with PowerShell installed.

TROUBLESHOOTING NOTE

If you encounter authentication errors: - Ensure the Proxmox server is reachable. - Verify that API access is enabled for the user. - Confirm the correct realm is specified (e.g., @pam, @pve). - For API tokens, use the format: USER@REALM!TOKENID=UUID.
